# Claude Code Automation Infrastructure Setup Prompt

이 프롬프트를 Claude Code에서 복사하여 사용하세요.

## 🚀 설치 프롬프트 (복사하여 사용)

```
이 프로젝트에 Claude Code 자동화 인프라를 설치하고 싶어.
참고 저장소: https://github.com/diet103/claude-code-infrastructure-showcase

프로젝트 정보:
- 언어: C# (.NET)
- 프레임워크: DotNetty, Protocol Buffers
- 구조: Visual Studio 솔루션
- 주요 컴포넌트:
  * GameServer: DotNetty 기반 게임 서버
  * NetworkCore: 네트워크 코어 라이브러리
  * Protocol: Protocol Buffer 메시지 정의
  * Session: 세션 및 연결 관리
  * Lobby: 로비 시스템
  * Room: 게임 룸 관리
- 현재 작업: DotNetty 채널 핸들러, Protocol Buffer 통신

다음 순서로 설치해줘:

Phase 1: 필수 훅 설치
- skill-activation-prompt 훅 (프롬프트 기반 스킬 자동 제안)
- post-tool-use-tracker 훅 (파일 변경 추적)
- .claude/settings.json 생성 및 설정

Phase 2: C# DotNetty 게임 서버 전용 스킬 생성
참고 저장소의 스킬은 TypeScript/React용이므로 새로 작성 필요.

스킬 이름: csharp-dotnetty-gameserver
포함 내용:
- DotNetty 아키텍처 패턴 (채널, 핸들러, 파이프라인)
- Protocol Buffer 직렬화/역직렬화 패턴
- 채널 핸들러 구현 (인바운드/아웃바운드)
- 세션 관리 베스트 프랙티스
- 로비/룸 시스템 패턴
- async/await 비동기 프로그래밍
- 스레드 안전성 (ConcurrentDictionary 등)
- IDisposable 리소스 관리 패턴
- 에러 핸들링 및 복원력
- 메모리 관리 및 성능 최적화

스킬 활성화 조건 (skill-rules.json):
- pathPatterns: ["**/*.cs", "**/*.proto", "**/Handlers/**/*.cs", "**/Session/**/*.cs"]
- keywords: ["dotnetty", "channel", "handler", "protobuf", "session", "lobby", "room", "async", "게임 서버", "채널", "핸들러"]

Phase 3: 범용 에이전트 복사
다음 에이전트들을 .claude/agents/에 복사하고 한글 설명 추가:
- code-architecture-reviewer (코드 아키텍처 리뷰)
- refactor-planner (리팩토링 계획 수립)
- documentation-architect (문서 자동 생성)
- plan-reviewer (개발 계획 검토)

각 에이전트에 C# DotNetty 게임 서버 특화 내용 추가:
- DotNetty 채널 핸들러 패턴 검증
- Protocol Buffer 사용 베스트 프랙티스
- async/await 및 Task 관리
- 스레드 안전성 및 동시성

Phase 4: Dev Docs 시스템
- dev/active/ 디렉토리 구조 생성
- /dev-docs 슬래시 커맨드 복사 및 설정 (한글 설명 추가)
- 현재 작업 문서화 템플릿 생성

Phase 5: 테스트 및 검증
- C# 파일 편집 시 csharp-dotnetty-gameserver 스킬 자동 활성화 확인
- "DotNetty 채널 핸들러 구현해줘" 프롬프트로 스킬 테스트
- "Protocol Buffer 메시지 정의 추가해줘" 프롬프트로 테스트
- "코드 리뷰해줘" 프롬프트로 에이전트 테스트
- 훅 동작 확인

제외 항목:
- backend-dev-guidelines (Express/Node.js 전용)
- frontend-dev-guidelines (React 전용)
- cpp-game-server-guidelines (C++ 전용)
- tsc-check.sh (TypeScript 전용)

각 Phase 완료 후 확인을 받고 다음으로 진행해줘.
```

## 📝 개별 작업 프롬프트

```

```


## 🎯 기대 효과

### 1. 자동 컨텍스트 활성화
- C# 파일 편집 시 DotNetty 가이드라인 자동 로드
- Protocol Buffer 파일 편집 시 메시지 정의 패턴 자동 로드
- 적절한 패턴과 베스트 프랙티스 즉시 적용

### 2. 코드 품질 향상
- 에이전트를 통한 자동 코드 리뷰
- 아키텍처 일관성 유지
- DotNetty 및 Protocol Buffer 베스트 프랙티스 준수

### 3. 리팩토링 체계화
- 단계별 리팩토링 계획 자동 생성
- 위험 요소 사전 식별
- 변경 영향 범위 분석

### 4. 문서화 자동화
- 개발 문서 자동 생성
- 컨텍스트 리셋 시에도 작업 연속성 유지
- 작업 히스토리 추적

### 5. 개발 속도 향상
- 반복적인 패턴 자동 적용
- DotNetty: 채널 핸들러 구조, 파이프라인 설정 자동 생성
- Protocol Buffers: 메시지 정의 및 직렬화 패턴 일관성 유지
- async/await: 비동기 프로그래밍 패턴 자동 적용

## 🆘 문제 해결

### 스킬이 자동 활성화 안 됨
```
skill-rules.json의 pathPatterns를 확인해줘.
현재 작업 중인 파일 경로와 일치하는지 체크해줘.
```

### 훅이 실행 안 됨
```
.claude/hooks/ 디렉토리의 스크립트들이 실행 가능한지 확인해줘.
chmod +x로 권한 설정이 필요할 수 있어.
```

### 에이전트 실행 실패
```
.claude/agents/ 디렉토리에 [에이전트명].md 파일이 있는지 확인해줘.
```
